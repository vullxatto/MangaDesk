# Требования к REST API бэкенда (MangaDesk)

Документ описывает контракты, необходимые для замены демо-данных на фронте. Формат каждого метода — по шаблону ниже.

---

## 1. Список глав (таблица «Главы», фильтры, сортировка)

| Поле | Значение |
| :--- | :--- |
| **HTTP Method** | GET |
| **URI** | `api/v1/chapters` |
| **Заголовки** | `Authorization: Bearer <access_token>`; опционально `X-Api-Key` / `apiKey` — по политике продукта |
| **Полномочия** | Авторизованный пользователь с правом просмотра глав в команде |
| **Параметры запроса** | `title` (string, опционально) — фильтр по названию проекта; `status` (enum: `ready`, `waiting_editor`, `edit`, `upload`, `ai`, опционально); `sort` (`date_desc` \| `date_asc` \| `number_desc` \| `number_asc`); `page`, `page_size` |
| **Тело запроса** | — |
| **Статус ответа** | 200 |
| **Модель ответа** | `{ "items": [ { "id": 1, "project_id": "uuid", "title": "Клинок, рассекающий демонов", "number": 106, "status": "waiting_editor", "updated_at": "2026-04-19T21:12:00Z", "editor": { "id": "uuid", "name": "Роберт" } \| null, "assigned_at": "2026-04-10T11:20:00Z" \| null } ], "total": 42 }` |
| **Ошибки** | **400 Bad Request**: некорректные query-параметры; **401 Unauthorized**: неверный или отсутствующий токен; **403 Forbidden**: нет доступа к команде |

---

## 2. Обновление метаданных главы (проект / номер, модалка по карандашу)

| Поле | Значение |
| :--- | :--- |
| **HTTP Method** | PATCH |
| **URI** | `api/v1/chapters/{chapter_id}` |
| **Заголовки** | `Authorization: Bearer <access_token>`; `Content-Type: application/json` |
| **Полномочия** | Пользователь с правом редактирования метаданных главы (роль в команде / владелец проекта) |
| **Параметры запроса** | — |
| **Тело запроса** | `{ "project_id": "uuid", "number": 107 }` — либо `title` + `number`, если проекты не нормализованы по id на бэке |
| **Статус ответа** | 200 |
| **Модель ответа** | `{ "id": 5, "project_id": "uuid", "title": "Клинок, рассекающий демонов", "number": 107, "status": "waiting_editor", "updated_at": "2026-04-19T22:00:00Z", "editor": null, "assigned_at": null }` |
| **Ошибки** | **400 Bad Request**: невалидное тело, номер < 1; **401 Unauthorized**: неверный или отсутствующий токен; **403 Forbidden**: нет прав; **404 Not Found**: глава не найдена; **409 Conflict**: глава с таким проектом и номером уже существует |

## 6. Стоимость обработки в токенах (модалка перед отправкой ZIP в обработку)

| Поле | Значение |
| :--- | :--- |
| **HTTP Method** | GET |
| **URI** | `api/v1/processing/quote` |
| **Заголовки** | `Authorization: Bearer <access_token>` |
| **Полномочия** | Авторизованный пользователь |
| **Параметры запроса** | `project_id` (uuid); `chapter_number` (int); опционально флаги `clean_sounds`, `type_sounds` (boolean) — чтобы вернуть итог с учётом выбранных опций |
| **Тело запроса** | — |
| **Статус ответа** | 200 |
| **Модель ответа** | `{ "base_translation_tokens": 840, "clean_sounds_tokens": 120, "type_sounds_tokens": 0, "total_tokens": 960 }` — тайп бесплатен: `type_sounds_tokens` всегда 0 в списании; `total` = base + (clean_sounds ? clean_sounds_tokens : 0) |
| **Ошибки** | **400 Bad Request**: невалидные параметры; **401 Unauthorized**; **404 Not Found**: проект не найден |

## 7. Отправка архива главы в обработку (ZIP + опции клин/тайп)

| Поле | Значение |
| :--- | :--- |
| **HTTP Method** | POST |
| **URI** | `api/v1/chapters/upload-and-process` |
| **Заголовки** | `Authorization: Bearer <access_token>`; `Content-Type: multipart/form-data` |
| **Полномочия** | Пользователь с правом загрузки глав |
| **Параметры запроса** | — |
| **Тело запроса** | `multipart`: поле `file` (ZIP); поля формы `project_id`, `chapter_number`, `editor_id` (опционально), `clean_sounds` (boolean), `type_sounds` (boolean) |
| **Статус ответа** | 202 |
| **Модель ответа** | `{ "job_id": "uuid", "chapter_id": 12 \| null, "status": "queued", "quoted_total_tokens": 960 }` |
| **Ошибки** | **400 Bad Request**: не ZIP, превышен размер, битый архив; **401 Unauthorized**; **403 Forbidden**; **409 Conflict**: номер главы для проекта уже занят; **413 Payload Too Large**: файл слишком большой; **422 Unprocessable Entity**: не хватает токенов на счёте |

## Примечания для бэкенда

- Версия в URI (`v1`) — пример; можно заменить на принятую у вас схему.
- Идентификаторы глав на фронте сейчас числовые (`id`); допустимо заменить на UUID — тогда обновить фронт и поля `chapter_id` везде.
- Для всех мутаций рекомендуется поддержка **идемпотентности** загрузки (например, заголовок `Idempotency-Key` на метод из п.7) — по желанию, отдельным подпунктом в спецификации.
